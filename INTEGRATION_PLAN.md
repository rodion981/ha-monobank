# План інтеграції Monobank для Home Assistant

## Аналіз даних API

### 1. Currency API (Курси валют)
**Endpoint:** `/bank/currency`

**Корисні дані:**
- `currencyCodeA` - код валюти A (ISO 4217)
- `currencyCodeB` - код валюти B (зазвичай 980 = UAH)
- `date` - timestamp оновлення курсу
- `rateBuy` - курс купівлі
- `rateSell` - курс продажу
- `rateCross` - крос-курс (для валют без прямої торгівлі)

**Використання в HA:**
- Сенсори для популярних валют (USD, EUR, GBP)
- Атрибути: курс купівлі, продажу, час оновлення
- Оновлення кожні 5 хвилин (обмеження API)

### 2. Client Info API (Інформація про клієнта)
**Endpoint:** `/personal/client-info`

**Корисні дані:**

#### Accounts (Рахунки):
- `id` - унікальний ID рахунку
- `sendId` - ID для переказів
- `currencyCode` - валюта рахунку (980=UAH, 840=USD, 978=EUR)
- `cashbackType` - тип кешбеку
- `balance` - баланс в копійках/центах
- `creditLimit` - кредитний ліміт
- `maskedPan` - маскований номер картки
- `type` - тип картки (black, white, eAid, diia, madeInUkraine)
- `iban` - IBAN рахунку

**Типи карток:**
- `black` - чорна картка (преміум)
- `white` - біла картка (стандартна)
- `eAid` - картка єПідтримка
- `diia` - картка Дія
- `madeInUkraine` - картка Made in Ukraine

#### Jars (Банки/Цілі):
- `id` - унікальний ID банки
- `sendId` - ID для переказів
- `title` - назва банки
- `description` - опис
- `currencyCode` - валюта
- `balance` - поточний баланс в копійках
- `goal` - ціль накопичення в копійках

**Використання в HA:**
- Окремий сенсор для кожного рахунку з балансом
- Окремий сенсор для кожної банки з прогресом до цілі
- Атрибути: тип картки, валюта, IBAN, маскований номер

### 3. Statement API (Виписка)
**Endpoint:** `/personal/statement/{account}/{from}/{to}`

**Корисні дані:**
- Історія транзакцій
- Можна використати для:
  - Останні транзакції (event/notification)
  - Статистика витрат
  - Категорії витрат

### 4. Bank Sync API
**Дані для синхронізації:**
- `serverKeyId` - ID ключа сервера
- `serverPubKey` - публічний ключ
- `serverTimeMsec` - час сервера

## Функціонал інтеграції

### Фаза 1: Базовий функціонал (MVP)
1. **Конфігурація через UI**
   - Введення API токену
   - Валідація токену
   - Збереження в config entry

2. **Сенсори рахунків**
   - Баланс кожного рахунку
   - Атрибути:
     - Валюта
     - Тип картки
     - Маскований номер
     - IBAN
     - Кредитний ліміт
     - Тип кешбеку

3. **Сенсори банок (jars)**
   - Баланс кожної банки
   - Прогрес до цілі (%)
   - Атрибути:
     - Назва
     - Опис
     - Ціль
     - Валюта

4. **Сенсори курсів валют**
   - USD/UAH (купівля/продаж)
   - EUR/UAH (купівля/продаж)
   - Атрибути:
     - Час оновлення
     - Курс купівлі
     - Курс продажу

### Фаза 2: Розширений функціонал
1. **Транзакції**
   - Event для нових транзакцій
   - Сенсор останньої транзакції
   - Атрибути транзакції

2. **Додаткові валюти**
   - Можливість вибору валют для відстеження
   - GBP, PLN, та інші популярні валюти

3. **Статистика**
   - Витрати за день/тиждень/місяць
   - Доходи за період
   - Категорії витрат

### Фаза 3: Додаткові можливості
1. **Webhook підтримка**
   - Реал-тайм оновлення при транзакціях
   - Миттєві сповіщення

2. **Сервіси**
   - Оновлення даних вручну
   - Отримання виписки за період

3. **Lovelace картки**
   - Кастомна картка для відображення рахунків
   - Картка з курсами валют

## Технічні деталі

### API обмеження:
- Максимум 60 запитів на хвилину
- Рекомендоване оновлення: кожні 60 секунд для рахунків
- Курси валют: кожні 5 хвилин

### Формат балансу:
- API повертає баланс в копійках/центах
- Потрібно ділити на 100 для відображення

### Валюти (ISO 4217):
- 980 = UAH (гривня)
- 840 = USD (долар)
- 978 = EUR (євро)
- 826 = GBP (фунт)

### Device Class для сенсорів:
- Баланс: `monetary`
- Курс валют: `monetary`
- Прогрес банки: `percentage` (для атрибуту)
